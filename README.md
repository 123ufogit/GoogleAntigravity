# GSI DEM & Vector Geospatial Converter

国土地理院の基盤地図情報（DEM、建築物の外周、道路縁・道路中心線）のXML/ZIPファイルを、GISソフトウェアや各種解析ツールで扱いやすい形式（GeoTIFF、Shapefile）へ変換するためのPythonツール群です。

本プロジェクトは、森林空間情報の解析や学術研究における空間データの再現性と共同利用（オープンサイエンス）の促進を目的に開発されています。

---

## ツール構成

本リポジトリは、目的別に2つのGUIアプリケーションに分かれています。

### 1. DEM（標高データ）変換ツール (`gsi_dem_converter.py`)
*   **用途**: 国土地理院 基盤地図情報のDEM（数値標高モデル）XML/ZIPファイルを、一括して地理座標系または平面直交座標系の **GeoTIFF (.tif)** に変換します。
*   **特徴**:
    *   複数メッシュの自動結合機能（シームレスな境界処理）。
    *   市区町村名CSVをロードして、特定の自治体のみをフィルタリング抽出可能。
    *   変換結果の図郭（インデックス）を Leaflet 地図でブラウザプレビューできる簡易ビューアを出力。

### 2. 道路・建物（ベクターデータ）結合ツール (`gsi_road_building_merger.py`)
*   **用途**: 基盤地図情報の基本項目に含まれる「建築物の外周（`BldA`）」、「道路縁（`RdEdg`）」、「道路中心線（`RdCL`）」のXML/ZIPファイルをパースし、市区町村ごとに結合した **ESRI Shapefile (.shp)** を出力します。
*   **特徴**:
    *   ファイル内の `fid`（地物ID）から市区町村コードを自動判別し、市区町村名ごとのShapefileにグループ化して出力。
    *   平面直交座標系（系1〜19）への自動/手動での投影変換と、投影定義ファイル `.prj` の自動出力。
    *   **道路中心線の自動抽出（Voronoi法）**: `RdCL` データが含まれない場合に、道路縁データ（`RdEdg`）からボロノイ分割を用いて近似的な中心線を自動抽出する高度な機能を搭載（※ Shapely依存、実験的機能）。

## インストールと環境構築

本ツール群は、C/C++バイナリに依存する空間情報ライブラリ（GDAL, GEOS, PROJ等）を使用します。動作確認済みの仮想環境を構築するために、**Conda (Anaconda / Miniconda)** を使用することを推奨します。

### Condaによる環境構築手順

1.  本リポジトリをダウンロード（またはクローン）します。
2.  ターミナル（Anaconda Promptなど）を開き、本リポジトリのディレクトリに移動します。
3.  以下のコマンドを実行して、`environment.yml` に定義された依存関係をインストールします。
    ```bash
    conda env create -f environment.yml
    ```
4.  作成された仮想環境を有効化します。
    ```bash
    conda activate gsi-gis-env
    ```

---

## 使用方法

仮想環境を有効化した状態で、以下のコマンドを実行してGUIを起動します。

**DEM（標高データ）の変換:**
```bash
python gsi_dem_converter.py
```

**道路・建物のベクターデータ変換:**
```bash
python gsi_road_building_merger.py
```

## 遵守すべき法令・データ利用上の注意

学術論文や共同研究、その他の成果物で本ツールを使用して変換したデータを使用する際は、以下の点に留意してください。

### 1. 国土地理院コンテンツ利用規約に基づく出典明記
本ツールで変換した国土地理院の基盤地図情報を利用した成果物を公開または第三者へ提供する場合、**出典の明記**が義務付けられています。
*   **表記例**:
    > 出典：国土地理院 基盤地図情報（数値標高モデル）
    > 出典：国土地理院 基盤地図情報（基本項目）

### 2. 測量法に基づく申請（複製・使用承認）
*   変換後のデータそのものをインターネット等で不特定多数へ二次配布・公開する場合は、**測量法に基づく承認申請**（複製承認または使用承認）が別途必要になる場合があります。
*   本ツールを他の研究者に紹介する際は、プログラムファイル（Pythonスクリプト）のみを共有するようにし、**変換元の生データや変換後の成果物データ自体を無断でリポジトリや公開の場に含めない**ように徹底してください。

### 3. 森林空間情報の取り扱いについて
*   森林簿や所有者境界データなど、個人情報や機密性の高い森林データと本ツールを組み合わせて処理を行う場合、共有リポジトリ（GitHubなど）にそれらの実データがコミットされないよう、`.gitignore` による除外が正しく動作していることを確認してください。
